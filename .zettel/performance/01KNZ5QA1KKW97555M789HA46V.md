Source: https://grafana.com/products/cloud/k6/ — Grafana Cloud k6 product page, fetched 2026-04-12.

Grafana Cloud k6 (formerly k6 Cloud, originally Load Impact Cloud) is the hosted, commercial version of k6 operated by Grafana Labs. It is k6 OSS plus managed infrastructure for large-scale distributed runs, plus the persistent dashboarding and analytics layer OSS k6 lacks.

## What you get over OSS

1. **Managed load generators** across 20+ global locations (US east/west, EU west/central, APAC, South America). Select regions per test to model geographically distributed user populations.
2. **Scale up to ~1M concurrent virtual users or ~5M RPS** per the documented ceilings. Impossible on a single OSS runner without serious engineering.
3. **Grafana k6 Studio** — a browser session recorder that converts real user flows into k6 scripts you can edit visually before running. This is the commercial answer to JMeter Recorder and Gatling Recorder.
4. **Cloud Insights** — automated anomaly detection on test results; highlights regressions vs a baseline without manual threshold tuning.
5. **Persistent test history** with run comparison, performance-over-time charts, and annotations. OSS k6 has no built-in history.
6. **Browser test artifacts** — screenshots, interaction timelines, Web Vitals traces captured per iteration of k6 browser tests.
7. **Correlate with Grafana Cloud observability** — profiles, logs, traces from the system under test sit next to the load-test result in the same dashboards.
8. **SLOs as pass/fail criteria** — thresholds become organisational SLOs with alerting.
9. **Scheduled test runs** as synthetic monitoring against production.

## CI integration

First-class GitHub Actions and GitLab CI integrations. `k6 cloud run script.js` from a CI step streams logs and returns the cloud run's exit code. Results link directly from the PR/MR.

## Pricing structure (as of source fetch)

- **Free**: 500 virtual-user-hours per month, community support, basic features.
- **Pro**: $0.15/VUH after free tier, $19/month platform fee, 8×5 email support.
- **Enterprise**: minimum $25,000/year, starting $0.05/VUH, premium support, SSO, deployment flexibility (including private load zones).

A "virtual user hour" is one VU running for one hour; a 30-minute test with 100 VUs is 50 VUH. Pricing is the dominant constraint for large soak or scheduled-monitoring use cases.

## Relationship to OSS k6

Grafana Cloud k6 runs the same open-source k6 binary. A script that runs `k6 run script.js` locally runs identically via `k6 cloud run script.js`. There is no vendor-specific syntax. This lowers switching cost: you can start local, promote to cloud, roll back to local. Contrast JMeter→BlazeMeter or Gatling→Gatling Enterprise, both of which have some commercial-only APIs.

## Strengths

- Zero script rewrite from OSS to cloud.
- Largest free tier in commercial load testing SaaS.
- Geographic distribution without provisioning.
- Native Grafana observability correlation.
- Browser testing with capture artifacts is unmatched among hosted load generators.

## Failure modes

- **VUH cost at the tail** — large enterprise tests quickly push into the Enterprise tier minimum.
- **Load zones are public cloud regions** — if your SUT is behind a strict network, you need private zones (Enterprise feature) or self-hosted k6-operator in your own infra.
- **Historical data retention** is tier-gated; raw results beyond a retention window must be exported.
- **"Managed" means you don't control the client machine** — debugging "why is my load generator slow" requires support tickets.

## Relevance to APEX G-46

Grafana Cloud k6 is the closest production-ready environment for running APEX-generated worst-case reproduction scripts at scale. The OSS/cloud binary compatibility means APEX can emit k6 scripts without worrying about which runtime they land on. For tail-latency regression detection over long horizons, Cloud's persistent history is the natural backing store — the comparison baselines G-46's regression detector needs already exist there.

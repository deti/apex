# AFL++ — American Fuzzy Lop plus plus

*Source: https://aflplus.plus/ — fetched 2026-04-10.*

## Description and History

AFL++ is the continuation of the American Fuzzy Lop fuzzer created by Michał "lcamtuf" Zalewski. It was established to "incorporate all the best features developed in the years for the fuzzers in the AFL family" that weren't integrated into the original AFL project after its last update in November 2017.

## Core Components

- **afl-fuzz** — a fuzzer supporting multiple mutators and configurations
- **Instrumentation modules** for source code — LLVM mode, afl-as, and GCC plugin
- **Binary instrumentation options** — QEMU mode, Unicorn mode, and QBDI mode
- **Utility programs** — afl-tmin and afl-cmin for corpus minimisation
- **Supporting libraries** — libtokencap, libdislocator, and libcompcov

## Key Enhancements over vanilla AFL

"A lot of changes, optimisations and new features" compared to original AFL, including:

- AFLfast power schedules (Böhme et al.)
- QEMU 5.1 with CompareCoverage
- MOpt mutators (optimised mutator scheduling)
- InsTrim instrumentation
- Persistent mode
- LAF-Intel splitting of comparisons
- CmpLog (comparison operand logging)
- Custom mutator API
- Grammar-aware mutators

## Notable Achievements

AFL++ has identified vulnerabilities across numerous applications including VLC (11 CVEs), SQLite, Vim, FreeRDP, FFmpeg, and PostgreSQL.

## Maintenance

Maintained by Marc Heuse, Heiko Eißfeldt, Andrea Fioraldi, and Dominik Maier. Active development on GitHub.

## Relevance to APEX G-46

AFL++ is the **baseline** against which any new fuzzing capability is judged. Its `-p exploit` scheduling mode can be coaxed toward resource-maximising behaviour, but it is not a full PerfFuzz replacement — the feedback function is still coverage-based. The G-46 spec explicitly notes: "AFL++ supports `-p exploit` mode which can be directed toward maximising execution time, and its CMPLOG instrumentation tracks comparison operands. However, AFL++ does not have a dedicated performance-fuzzing mode."

APEX's resource-guided fuzzer should be benchmarked against AFL++ `-p exploit` on the PerfFuzz benchmark suite to quantify the gain from true resource-maximising feedback.
